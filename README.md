# ห้องเกม — สไลซ์ที่ 1

หน้าแรก (ชื่อ / Host / Join) + ห้องรอ (รายชื่อ / Host / Ready / Start)
ยังไม่มีเกม ยังไม่มีแชท ปุ่ม Setting ปิดไว้

## ต้องตั้งค่าใน Firebase ก่อน 3 อย่าง

1. **Authentication → Sign-in method → Anonymous → Enable**
   ถ้าไม่เปิด หน้าเว็บจะขึ้นข้อความบอกตรง ๆ ว่าต้องไปเปิด
2. **Firestore Database → Create database** (Production mode)
3. **Firestore → Rules** → วางไฟล์ `firestore.rules` → Publish

เมื่อ push ขึ้น GitHub Pages แล้ว เพิ่มโดเมนที่
**Authentication → Settings → Authorized domains** → `j1gsaww.github.io`
ไม่งั้นจะขึ้น `auth/unauthorized-domain`

## ทดสอบ

เปิด **สองเบราว์เซอร์คนละยี่ห้อ** หรือ **หน้าต่างปกติ + หน้าต่างส่วนตัว**

> ⚠️ สองแท็บในเบราว์เซอร์เดียวกันจะได้รหัสประจำตัวเดียวกัน
> ระบบจะมองว่าเป็นคนเดียว ไม่ใช่สองผู้เล่น

| ทดสอบ | คาดหวัง |
|---|---|
| ฝั่ง A กด Host | ได้รหัส 4 ตัว เห็นชื่อตัวเองพร้อมป้าย Host |
| ฝั่ง B กรอกรหัส กด Join | ทั้งสองฝั่งเห็นรายชื่อครบภายใน 1–2 วินาที |
| กด Ready สลับไปมา | อีกฝั่งเห็นป้ายเปลี่ยนทันที |
| A กด Start ตอน B ยังไม่ Ready | ปุ่มเทา กดไม่ได้ ข้อความบอกว่ารออีกกี่คน |
| ทั้งคู่ Ready แล้ว A กด Start | ทั้งสองฝั่งเปลี่ยนหน้าพร้อมกัน |
| **ปิดหน้าต่าง A ทิ้ง** | B ได้เป็น Host แทนเกือบทันที (ดูข้อความใน console) |
| **ดึงสายเน็ตของ A** | B ได้เป็น Host แทนภายใน ~25 วินาที |
| B ออกจากห้องคนสุดท้าย | ห้องถูกลบออกจาก Firestore |

## สิ่งที่วางไว้แล้วแม้ยังไม่ได้ใช้

- **ไม่มีสถานะใดอยู่ในหน่วยความจำของเครื่องไหน** ทุกอย่างอ่านจาก Firestore ใหม่ทุกครั้ง
  เจ้าของห้องจึงเปลี่ยนตัวได้กลางคันโดยไม่ต้องส่งต่ออะไร
- **การโอนตำแหน่งเจ้าของห้อง** ทำงานแล้ว ทั้งฝั่งโค้ดและฝั่ง Security Rules
- **`role` / `seat`** มีอยู่ในข้อมูลแล้ว คนที่เข้าห้องตอนสถานะ playing จะเป็นคนดูอัตโนมัติ
- **`seq`** เพิ่มขึ้นทุกครั้งที่สถานะห้องเปลี่ยน เตรียมไว้กันคำสั่งซ้ำตอนมีลูป action
- **เพดาน 15 คน** บังคับตอน Join แล้ว
