# ห้องเกม — สไลซ์ที่ 1

หน้าแรก (ชื่อ / Host / Join / Setting) + ห้องรอ (รายชื่อ / Host / Ready / Start)
Setting มีสลับภาษาไทย–อังกฤษแล้ว ยังไม่มีเกม ยังไม่มีแชท

## ต้องตั้งค่าใน Firebase ก่อน 3 อย่าง

1. **Authentication → Sign-in method → Anonymous → Enable**
   ถ้าไม่เปิด หน้าเว็บจะขึ้นข้อความบอกตรง ๆ ว่าต้องไปเปิด
2. **Firestore Database → Create database** (Production mode)
3. **Firestore → Rules** → วางไฟล์ `firestore.rules` → Publish

เมื่อ push ขึ้น GitHub Pages แล้ว เพิ่มโดเมนที่
**Authentication → Settings → Authorized domains** → `j1gsaww.github.io`
ไม่งั้นจะขึ้น `auth/unauthorized-domain`

## ทดสอบ

เปิด **สองเบราว์เซอร์คนละยี่ห้อ** หรือ **หน้าต่างปกติ + หน้าต่างส่วนตัว**

> ⚠️ สองแท็บในเบราว์เซอร์เดียวกันจะได้รหัสประจำตัวเดียวกัน
> ระบบจะมองว่าเป็นคนเดียว ไม่ใช่สองผู้เล่น

**เทสหลายคนในเบราว์เซอร์เดียว** — เติม `?dev=multi` ท้าย URL
แต่ละแท็บจะได้ผู้เล่นของตัวเอง เปิด 4 แท็บเทส 4 คนได้เลย
ข้อแลก: แท็บนั้นรีเฟรชแล้วกลายเป็นคนใหม่ทันที ห้ามใช้ตอนเทสเรื่องกลับเข้าที่นั่งเดิม

| ทดสอบ | คาดหวัง |
|---|---|
| ฝั่ง A กด Host | ได้รหัส 4 ตัว เห็นชื่อตัวเองพร้อมป้าย Host |
| ฝั่ง B กรอกรหัส กด Join | ทั้งสองฝั่งเห็นรายชื่อครบภายใน 1–2 วินาที |
| กด Ready สลับไปมา | อีกฝั่งเห็นป้ายเปลี่ยนทันที |
| A กด Start ตอน B ยังไม่ Ready | ปุ่มเทา กดไม่ได้ ข้อความบอกว่ารออีกกี่คน |
| ทั้งคู่ Ready แล้ว A กด Start | ทั้งสองฝั่งเปลี่ยนหน้าพร้อมกัน |
| **ปิดหน้าต่าง A ทิ้ง** | B ได้เป็น Host แทนเกือบทันที (ดูข้อความใน console) |
| **ดึงสายเน็ตของ A** | B ได้เป็น Host แทนภายใน ~25 วินาที |
| B ออกจากห้องคนสุดท้าย | ห้องถูกลบออกจาก Firestore |

## เพลงพื้นหลัง

วางไฟล์ที่ `assets/music/Lobby Music.mp3` ชื่อต้องตรงเป๊ะ
ที่อยู่ไฟล์อยู่ใน `js/env.js` บรรทัดเดียว (`MUSIC_SRC`) เปลี่ยนชื่อไฟล์แล้วแก้ที่นั่น

**วิธีที่ใช้: เริ่มเล่นแบบปิดเสียง แล้วค่อยเปิดเสียงตอนแตะครั้งแรก**

เบราว์เซอร์ห้ามเล่น *เสียง* เอง แต่ไม่เคยห้ามเล่นสื่อที่ปิดเสียงไว้
เพลงจึงเดินตั้งแต่วินาทีแรกจริง ๆ แค่ยังไม่ได้ยิน พอผู้ใช้คลิกอะไรสักอย่าง
เราแค่เลิกปิดเสียง ไม่ได้เริ่มเล่นใหม่ เสียงจึงมาทันทีไม่มีสะดุดและไม่ต้องรอโหลด

`navigator.getAutoplayPolicy()` ใช้เช็กล่วงหน้าว่าเว็บนี้ได้รับอนุญาตแล้วหรือยัง
ถ้าได้แล้วก็ข้ามขั้นปิดเสียงไปเลย (Chrome/Firefox มี Safari ยังไม่มี จึงเช็กก่อนเรียก)

ไม่มี API ให้เว็บ "ขออนุญาตเล่นเสียง" แบบป๊อปอัพ — Chrome ตัดสินจากพฤติกรรมการใช้เว็บ
ส่วน Firefox / Safari / Edge ให้ผู้ใช้ตั้งค่าเองรายเว็บได้ แต่พึ่งไม่ได้ เพราะเพื่อนทุกคน
จะเจอสถานะเริ่มต้นเหมือนกันหมด

แถบปรับเสียงอยู่ท้ายหน้าห้อง · ระดับเสียงกับสถานะปิดเสียงเก็บใน localStorage
ลากลงสุด = ปิดเสียงอัตโนมัติ · เพลงวนซ้ำและค่อย ๆ ไล่ระดับตอนเริ่ม ไม่ตูมใส่หน้า

ถ้าหาไฟล์ไม่เจอ จะขึ้นข้อความสีแดงบอกที่อยู่ที่หาไม่เจอ แทนที่จะเงียบเฉย ๆ

## เสียงประกอบ

```
assets/effect/Card.mp3     ลงไพ่ทุกครั้ง
assets/effect/Magic.mp3    ลงตองหรือโฟร์ (ซ้อนกับ Card.mp3 หน่วง 110ms)
assets/effect/Fairy.mp3    ทิศการเล่นกลับด้าน (เข้าเงื่อนไขข้อ 7)
assets/music/slave.mp3     เพลงพื้นหลังระหว่างเล่นสลาฟ
```

เสียงประกอบดังที่เครื่องทุกคน ไม่ใช่เฉพาะคนที่กด เพราะ `sound.js` ดูจาก
**สถานะที่เปลี่ยนไป** ไม่ได้ผูกกับปุ่ม คนที่รีเฟรชกลับเข้ามาจึงได้ยินตรงกับคนอื่น
และครั้งแรกที่เห็นสถานะจะเงียบไว้ก่อน ไม่เล่นเสียงย้อนหลังรัวใส่

ระดับเสียงใช้แถบเดียวกับเพลง กดปิดเสียงแล้วเงียบหมดทั้งคู่
แต่ตั้งฐานเอฟเฟกต์ให้ดังกว่าเพลง 1.5 เท่า เพราะเสียงสั้นต้องแทรกเพลงขึ้นมาได้

เพลงพื้นหลังสลับตามเกม — เกมประกาศ `music` ในทะเบียนของตัวเอง
เข้าเกมแล้วหรี่เพลงห้องลงแล้วสลับ ออกจากเกมแล้วกลับเพลงเดิม

## ภาษา

ข้อความทุกคำอยู่ใน `js/i18n.js` ที่เดียว ไฟล์อื่นถืออ้างได้แค่คีย์

```
ใน HTML   <button data-i18n="home.host"></button>
          <input data-i18n-placeholder="home.namePlaceholder">
          <button data-i18n-title="lobby.copy">
ใน JS     t('lobby.waitReady', { n: 3 })
ในข้อผิดพลาด   throw new AppError('err.roomFull')
```

ข้อผิดพลาดถือ **คีย์** ไม่ใช่ข้อความ จึงเปลี่ยนภาษาได้ทั้งที่ข้อความค้างอยู่บนจอ

เพิ่มภาษาใหม่ = เติมก้อนใหม่ใน `DICT` แล้วเติมชื่อใน `LANGS` ปุ่มเลือกภาษาสร้างจาก `LANGS` เอง ไม่ต้องแก้ HTML

พิมพ์ `__i18nAudit()` ใน console เพื่อดูว่ามีคีย์ไหนแปลไม่ครบ

ภาษาเก็บใน localStorage ไม่ขึ้น Firestore — คนละคนในห้องเดียวกันเลือกคนละภาษาได้
ครั้งแรกที่เปิดจะเดาจากภาษาของเบราว์เซอร์

## สิ่งที่วางไว้แล้วแม้ยังไม่ได้ใช้

- **ไม่มีสถานะใดอยู่ในหน่วยความจำของเครื่องไหน** ทุกอย่างอ่านจาก Firestore ใหม่ทุกครั้ง
  เจ้าของห้องจึงเปลี่ยนตัวได้กลางคันโดยไม่ต้องส่งต่ออะไร
- **การโอนตำแหน่งเจ้าของห้อง** ทำงานแล้ว ทั้งฝั่งโค้ดและฝั่ง Security Rules
- **`role` / `seat`** มีอยู่ในข้อมูลแล้ว คนที่เข้าห้องตอนสถานะ playing จะเป็นคนดูอัตโนมัติ
- **`seq`** เพิ่มขึ้นทุกครั้งที่สถานะห้องเปลี่ยน เตรียมไว้กันคำสั่งซ้ำตอนมีลูป action
- **เพดาน 15 คน** บังคับตอน Join แล้ว

## เวลาแก้แล้วหน้าเว็บไม่เปลี่ยน

ทุกไฟล์มีรหัสรุ่นประทับไว้ — `data-build` บน `<body>` และ `window.BUILD` ใน `js/env.js`
ตอนบูตแอปจะเทียบสองค่านี้ ถ้าไม่ตรงแปลว่ามีไฟล์ค้างแคช จะขึ้นแถบแดงล่างจอ
พร้อมบอกว่าฝั่งไหนเก่า

`css/app.css` และ `js/*.js` ต่อท้ายด้วย `?v=<รุ่น>` แล้ว จึงไม่ค้างแคช
แต่ **`index.html` เติมอะไรไม่ได้** GitHub Pages แคชไว้ราว 10 นาที
ถ้าแก้ผังหน้าจอแล้วไม่เห็นผล ให้ Ctrl+Shift+R ก่อนเสมอ

**เปลี่ยนรหัสรุ่นทุกครั้งที่แก้ไฟล์** — แก้ที่ `data-build`, ทั้งสาม `?v=` และ `window.BUILD`
ให้เป็นค่าเดียวกัน

## ทะเบียนเกม

`js/games.js` — เกมพกของตัวเองมาทั้งหมด: ชื่อ คำอธิบาย รูปปก ภาพโต๊ะ จำนวนคน
ตัวเลือกในหน้าตั้งค่า และคำแปลของตัวเอง หน้าเลือกเกมวาดจากข้อมูลพวกนี้ล้วน ๆ
ไม่รู้จักเกมไหนเป็นการเฉพาะ

เพิ่มเกมใหม่ = สร้างโฟลเดอร์ใน `js/games/` แล้วเติมหนึ่งบรรทัดใน `js/games/index.js`

คีย์คำแปลตั้งชื่อตามระเบียบ ไม่ต้องประกาศซ้ำ

```
game.<id>.name                ชื่อเกม
game.<id>.desc                คำอธิบาย
game.<id>.<setting>           ชื่อตัวเลือก
game.<id>.<setting>.<value>   ชื่อค่าของตัวเลือก
```

ไฟล์ภาพที่เกมสลาฟใช้

```
assets/game/slave/cover.png   รูปปกในหน้าเลือกเกม
assets/game/slave/table.png   พื้นหลังหน้าเกม
```

หาไฟล์ไม่เจอจะขึ้นการ์ดเปล่ามีเครื่องหมายคำถามแทน ไม่พังทั้งหน้า
